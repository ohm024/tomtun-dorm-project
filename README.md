## 📑 2.2 เอกสารความต้องการของระบบ (SRS)
เอกสารข้อกำหนดความต้องการซอฟต์แวร์ (Software Requirements Specification: SRS)
โครงการ: ระบบจัดการห้องพักรายเดือนเวอร์ชัน: 1.2 (ฉบับสมบูรณ์ ผสานโครงสร้างและรายละเอียด)บันทึกการปรับแก้ไขเอกสาร (Revision History)เวอร์ชันวันที่รายละเอียดการปรับแก้ไข1.0-เอกสารต้นฉบับ ระบุรายละเอียดฟังก์ชันการทำงานระดับปฏิบัติการ1.1-ปรับโครงสร้างตามมาตรฐาน IEEE 830, เพิ่มขอบเขต, Actor, ข้อสมมติ และข้อจำกัด1.2ปัจจุบันผสานรายละเอียดฟังก์ชันยิบย่อยจาก 1.0 เข้ากับโครงสร้างมาตรฐานของ 1.1 ให้เป็นฉบับสมบูรณ์
1.บทนำ (Introduction)
1.1 วัตถุประสงค์ของเอกสารเอกสารนี้อธิบายข้อกำหนดความต้องการซอฟต์แวร์สำหรับระบบจัดการห้องพักรายเดือน เพื่อใช้เป็นแนวทางในการพัฒนา ทดสอบ ส่งมอบ และบำรุงรักษาระบบ โดยเป็นเอกสารอ้างอิงร่วมกันระหว่างผู้ว่าจ้าง ผู้พัฒนา และผู้มีส่วนเกี่ยวข้อง
1.2 ขอบเขตของระบบ (System Scope)ระบบจัดการห้องพักรายเดือนเป็น Web-based Application สำหรับเจ้าของอพาร์ตเมนต์/หอพัก ใช้บริหารจัดการข้อมูลตั้งแต่การจัดการห้องพัก การจอง การทำสัญญา รับผู้เช่าเข้าพักและย้ายออก การจัดเก็บค่าเช่าและค่าสาธารณูปโภค ไปจนถึงการออกรายงานและ Dashboard เชิงวิเคราะห์1.3 ผู้ใช้งานระบบ (Stakeholders & Users)เจ้าของกิจการ / ผู้บริหาร: ดู Dashboard ภาพรวม, รายงานการเงิน, และอนุมัติรายการสำคัญผู้จัดการอาคาร: จัดการข้อมูลห้องพัก, สัญญาเช่า, และดูแลภาพรวมการดำเนินงานเจ้าหน้าที่การเงิน: จัดการใบแจ้งหนี้, รับชำระเงิน, คืนเงินประกัน, และติดตามหนี้ค้างชำระเจ้าหน้าที่ดูแลอาคาร (นิติบุคคล): จดมิเตอร์น้ำ-ไฟ, บันทึกการแจ้งซ่อม, และตรวจสอบสภาพห้องผู้เช่า: ดูข้อมูลสัญญา, ใบแจ้งหนี้, ประวัติการชำระเงิน, และแจ้งซ่อม (จำกัดสิทธิ์การเข้าถึง)
1.4 คำจำกัดความ (Definitions)ผู้เช่า: บุคคลที่ทำสัญญาเช่าห้องพักรายเดือนOccupancy Rate: อัตราส่วนห้องที่มีผู้เช่าเทียบกับห้องทั้งหมดInvoice: ใบแจ้งหนี้ค่าเช่าและค่าบริการประจำเดือน2. ภาพรวมของระบบ (Overall Description
2.1 มุมมองระบบ (System Perspective)ระบบเป็น Web-based Application เชื่อมต่อฐานข้อมูลส่วนกลาง (Cloud-based) สามารถใช้งานผ่านเว็บเบราว์เซอร์ได้ทั้งบนคอมพิวเตอร์และอุปกรณ์มือถือ (Responsive Design
2.2 ข้อสมมติ (Assumptions)ผู้ใช้งานมีความรู้พื้นฐานด้านการใช้งานคอมพิวเตอร์และสมาร์ทโฟนอุปกรณ์ที่ใช้งานมีการเชื่อมต่ออินเทอร์เน็ตตลอดเวลาที่ทำรายการ
2.3 ข้อจำกัด (Constraints)กฎหมาย: ต้องปฏิบัติตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (PDPA)การเก็บข้อมูล: ระบบต้องสามารถเก็บข้อมูลย้อนหลังได้สูงสุด 7 ปีเทคนิค: รองรับการทำงานบน Web Browser รุ่นใหม่ (Chrome, Edge, Safari, Firefox
3. ความต้องการด้านฟังก์ชันการทำงาน (Functional Requirements)
3.1 การจัดการห้องพักและสถานะ (Room Management)การจัดการข้อมูลห้องพัก:เพิ่มห้องพักใหม่: กำหนดหมายเลขห้อง, ชั้น, ประเภทห้อง (ห้องเปล่า, เฟอร์นิเจอร์บางส่วน, เฟอร์นิเจอร์ครบ), ขนาดห้อง (ตร.ม.), สิ่งอำนวยความสะดวกพื้นฐาน, อัพโหลดรูปภาพ, กำหนดอัตราค่าเช่า/ค่าประกัน, และกำหนดมิเตอร์น้ำ-ไฟประจำห้องแก้ไขข้อมูล: ปรับปรุงรายละเอียด, บันทึกประวัติการซ่อมแซม, อัพเดทรายการทรัพย์สินในห้อง, และบันทึกการเปลี่ยนแปลงอัตราค่าเช่าลบข้อมูล (Soft Delete): ตรวจสอบประวัติการเช่าก่อนลบ, ย้ายข้อมูลลงถังขยะชั่วคราว และกู้คืนได้ภายใน 30 วันการจัดการสถานะห้องพัก:แสดงสถานะ: ว่าง (พร้อมให้เช่า), จอง (มีการจองล่วงหน้า), มีผู้เช่า, ซ่อมแซม, ปิดปรับปรุงการทำงานอัตโนมัติ: อัพเดทสถานะอัตโนมัติตามสัญญาเช่า, บันทึกการแจ้งซ่อมและติดตามระยะเวลา, พร้อมแจ้งเตือนเมื่อใกล้ครบกำหนดสัญญา3.2 การจองและสัญญาเช่า (Booking & Contract)การจองห้องพัก:ตรวจสอบห้องว่าง, บันทึกข้อมูลผู้จอง, กำหนดวันเข้าพัก, รับเงินจอง/มัดจำ, ออกใบจอง, และกำหนดระยะเวลาจองก่อนทำสัญญายกเลิกการจอง: ตรวจสอบเงื่อนไข คืนเงินมัดจำ และปรับสถานะห้องเป็น "ว่าง" อัตโนมัติการทำสัญญาเช่า:สร้างใหม่: บันทึกข้อมูลผู้เช่าและผู้ค้ำประกัน, กำหนดระยะเวลา, อัตราค่าเช่า, ค่าประกัน, เงื่อนไขพิเศษ และอัพโหลดไฟล์เอกสารประกอบสัญญาการต่ออายุ: แจ้งเตือนก่อนหมดสัญญา, ปรับปรุงข้อมูล/อัตราค่าเช่า, และออกสัญญาฉบับใหม่
3.3 การรับผู้เช่าเข้าพักและย้ายออก (Check-in & Check-out)การรับเข้าพัก (Check-in):ตรวจสอบเอกสาร, บันทึกค่ามิเตอร์น้ำ-ไฟเริ่มต้น, ส่งมอบกุญแจ/คีย์การ์ด, ชี้แจงกฎระเบียบตรวจสอบสภาพห้องและทรัพย์สิน ถ่ายรูปเก็บเป็นหลักฐานก่อนเข้าพักการย้ายออก (Check-out):นัดหมายตรวจสอบห้อง, บันทึกความเสียหาย, อ่านมิเตอร์น้ำ-ไฟครั้งสุดท้าย, คำนวณค่าใช้จ่ายคงค้างการคืนเงินประกัน: ตรวจสอบเงื่อนไข, หักค่าเสียหาย (ถ้ามี), คำนวณยอดสุทธิ และออกเอกสารการคืนเงิน
3.4 การจัดการผู้เช่า (Tenant Management)จัดเก็บข้อมูล: ข้อมูลส่วนตัว, ข้อมูลติดต่อ, เอกสารประจำตัว, ข้อมูลยานพาหนะ, และผู้ติดต่อฉุกเฉินประวัติ: อัพเดตข้อมูลผู้เช่า, บันทึกประวัติการเช่า, พฤติกรรมผู้เช่า, และระบบจัดการข้อร้องเรียน (Complaint Management
)3.5 การจัดเก็บค่าเช่าและค่าบริการ (Billing & Utilities)การจัดการค่าเช่า:สร้างใบแจ้งหนี้ (Invoice) ประจำเดือนอัตโนมัติ, คำนวณค่าน้ำ-ไฟ, รวมค่าบริการเสริมกำหนดวันครบกำหนดชำระ, ส่งใบแจ้งหนี้อัตโนมัติรับชำระผ่านหลายช่องทาง, บันทึกการชำระเงิน, ออกใบเสร็จรับเงิน, และมีระบบติดตามสถานะการค้างชำระการจัดการค่าสาธารณูปโภค (มิเตอร์):บันทึกค่ามิเตอร์ประจำเดือน, คำนวณค่าน้ำ-ไฟอัตโนมัติ, แจ้งเตือนกรณีมีการใช้งานผิดปกติคำนวณค่าส่วนกลาง, บันทึกค่าใช้จ่ายพิเศษ, และแยกประเภทค่าบริการได้
3.6 ระบบ Dashboard และการออกรายงาน (Dashboard & Reporting)Dashboard ภาพรวม:แสดงจำนวนห้องพักทั้งหมดแยกตามสถานะ (ว่าง, จอง, มีผู้เช่า, ซ่อมแซม)แสดงอัตราการเช่า (Occupancy Rate)สรุปการเงิน: รายได้ค่าเช่า, ค่าน้ำ-ไฟ, ค่าบริการ, ยอดค้างชำระรวม, เงินประกันรวม และกราฟรายได้ย้อนหลัง 12 เดือนแจ้งเตือนสำคัญ: สัญญาใกล้หมดอายุ, ค้างชำระ, แจ้งซ่อมรอดำเนินการ, นัดหมายตรวจห้องDashboard ด้านการเงิน & บริหาร:สถานะชำระเงินรายห้อง, กราฟการชำระตรงเวลา/ล่าช้า, รายการค้างชำระตามอายุหนี้วิเคราะห์รายได้, ต้นทุน, กำไร, สถานะการซ่อมบำรุง, และวิเคราะห์ข้อมูลผู้เช่า (ระยะเวลาเช่าเฉลี่ย, สาเหตุย้ายออก)การออกรายงาน (Reports):รายงานการเช่า: อัตราการเช่า, การต่อสัญญา, การย้ายออกรายงานการเงิน: รายได้ประจำเดือน, ลูกหนี้ค้างชำระ, ภาษีหัก ณ ที่จ่าย, การคืนเงินประกันรายงานซ่อมบำรุง: ประวัติการซ่อม, ค่าใช้จ่ายการซ่อมบำรุง
4. ความต้องการที่ไม่ใช่ฟังก์ชันการทำงาน (Non-Functional Requirements)
4.1 ประสิทธิภาพ (Performance)ระบบต้องรองรับผู้ใช้งานพร้อมกัน (Concurrent Users) ได้อย่างน้อย 50 คนโดยไม่หน่วงเวลาในการตอบสนอง (Response Time) ของระบบต้องไม่เกิน 3 วินาทีต่อการทำรายการระบบต้องสามารถทำงานและเข้าถึงได้ตลอด 24 ชั่วโมง (High Availability)
4.2 ความปลอดภัย (Security)Authentication & Authorization: มีระบบยืนยันตัวตน และจัดการสิทธิ์การเข้าถึงข้อมูลตามบทบาทผู้ใช้ (Role-based Access Control)Data Encryption: เข้ารหัสข้อมูลส่วนบุคคลและข้อมูลรหัสผ่านตามมาตรฐานความปลอดภัยAudit Log: มีระบบบันทึกประวัติการใช้งานระบบ (ใคร ทำอะไร เมื่อไหร่)Backup: มีระบบสำรองข้อมูล (Automated Backup) อย่างสม่ำเสมอ4.3 ความสามารถในการใช้งาน (Usability)User Interface (UI) ออกแบบให้ใช้งานง่าย และรองรับภาษาไทยเป็นหลักออกแบบระบบเป็น Responsive Web Design ให้แสดงผลได้ดีบนหน้าจอทุกขนาด (PC, Tablet, Mobile)มีคู่มือการใช้งาน (User Manual) และระบบช่วยเหลือออนไลน์ (Online Help)
